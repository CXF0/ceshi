# 项目全盘检查：Bug 与下一步迭代工作清单

## 一、当前高优先级 Bug（建议 1~2 周内完成）

### 1) 安全配置硬编码（严重）
- [ ] **移除硬编码数据库密码**：`server/src/app.module.ts` 中直接写了数据库账号密码，存在泄漏风险。
- [ ] **移除硬编码 JWT Secret**：`server/src/auth/auth.module.ts` 与 `server/src/auth/jwt.strategy.ts` 中写死了 secret。
- [ ] **改为环境变量注入**：引入 `.env.example`，并在 README 补齐部署说明。
- [ ] **上线前轮换密钥**：数据库密码/JWT secret 全量轮换，已发 token 失效策略一并明确。

### 2) CORS 过宽（高）
- [ ] `server/src/main.ts` 目前 `app.enableCors()` 未限制来源，应改为白名单域名。
- [ ] 静态资源 `/static/` 头部当前设置 `Access-Control-Allow-Origin: *`，需按环境限制。

### 3) 生产调试日志泄漏风险（中高）
- [ ] 前端多处 `console.log`（如 `web/src/pages/system/certification/index.tsx`）包含请求参数与 URL，可能暴露敏感信息。
- [ ] 后端启动日志输出完整服务地址与目录信息，生产环境需降级为结构化日志并脱敏。

### 4) 异常处理与类型约束不足（中高）
- [ ] 后端大量 `any` 入参，缺少 DTO + class-validator，易导致脏数据入库。
- [ ] 关键模块（合同、CRM、通知）补齐参数校验与统一异常返回格式。

### 5) 上传内容治理不足（中）
- [ ] 上传文件类型与大小策略需统一（当前上传入口分散）。
- [ ] 建议增加：扩展名 + MIME 双重校验、病毒扫描（可异步）、下载鉴权策略。

## 二、中优先级工程问题（建议 2~4 周）

### 6) 仓库包含运行期上传文件（中）
- [ ] `server/uploads/` 下存在大量业务文件，需立即从 Git 追踪中移除。
- [ ] 增加 `.gitignore` 规则：忽略运行期上传目录与构建产物。
- [ ] 清理历史大文件（如有必要，使用 git filter-repo）。

### 7) 前后端文档不足（中）
- [ ] 当前 `server/README.md` 与 `web/README.md` 基本仍是模板，需补齐真实业务文档。
- [ ] 增加模块边界、角色权限矩阵、部署拓扑、常见故障排查。

### 8) API 规范一致性不足（中）
- [ ] 统一分页结构与错误码。
- [ ] 统一字段命名风格（camel/snake），减少前端二次映射。
- [ ] 输出 OpenAPI/Swagger 文档，并作为联调基线。

### 9) 缺少自动化测试基线（中）
- [ ] 后端：至少为 auth/contract/crm 增加 service + controller 单测。
- [ ] 前端：关键页面（登录、合同列表、证书管理）增加组件/集成测试。
- [ ] 增加最小 e2e（登录->创建客户->创建合同）。

## 三、下一步迭代功能清单（按业务价值排序）

### A. 权限与审计（建议优先）
- [ ] 细粒度权限点（按钮级）统一配置中心，前后端同源。
- [ ] 审计日志：谁在何时修改了什么（合同状态、客户信息、通知发布）。
- [ ] 高风险操作二次确认（删除、状态回退、批量导入）。

### B. 合同与回款闭环
- [ ] 合同生命周期看板（待提交/审核中/执行中/已归档）。
- [ ] 回款提醒（按到期日、超期天数）+ 消息中心联动。
- [ ] 发票与附件统一归档视图，支持批量导出。

### C. CRM 精细化运营
- [ ] 客户分层标签（行业、规模、合作阶段、跟进热度）。
- [ ] 跟进记录时间轴 + 下次跟进提醒。
- [ ] 渠道转化漏斗报表（线索->商机->签约）。

### D. 仪表盘升级
- [ ] 管理层视图：收入目标达成率、回款预测、风险客户。
- [ ] 顾问视图：本周待办、到期证书、逾期合同。
- [ ] 支持自定义日期区间与数据导出。

### E. 通知系统增强
- [ ] 通知已读/未读分层策略 + 强提醒（站内+邮件/企业微信可选）。
- [ ] 富文本通知安全过滤（XSS 白名单）。
- [ ] 通知对象支持组织树批量选择与去重预览。

### F. 官网/内容运营能力
- [ ] 官网案例、服务项支持草稿/发布流。
- [ ] 图片素材中心（复用、压缩、格式转换）。
- [ ] SEO 基础能力：meta、sitemap、结构化数据。
